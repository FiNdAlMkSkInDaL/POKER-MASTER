# Project Overview: Local AI Poker Tutor & Simulator

## The Premise
We are building a locally hosted No-Limit Texas Hold'em simulator designed to coach the user and improve their fundamentals. The core mechanic is that the user must articulate their reasoning for their actions (e.g., "I am calling because I have the pot odds to hit my flush"). Behind the scenes, the game state and the user's reasoning are packaged into a prompt and sent to an LLM (Large Language Model), which acts as an expert poker coach to critique the play and correct mistakes.

This is a **two-phase project**:
* **Phase 1 (Current):** Build the core game engine, the User Interface, and integrate the LLM for real-time conversational feedback based on poker fundamentals.
* **Phase 2 (Future):** Integrate the GTO+ post-flop solver via a local socket connection to provide mathematically perfect post-flop tree analysis. 

## Tech Stack
* **Language:** Python 3.x
* **UI Framework:** Streamlit (or Gradio) for rapid prototyping of a web interface that can handle both the game display and a chat window.
* **Game Engine:** `clubs` (fschlatt/clubs), `RLCard`, or `PyPokerEngine` to handle deck logic, hand evaluation, betting rounds, and pot management.
* **LLM Integration:** Ollama (for running local models) or a standard API wrapper (OpenAI/Gemini) for cloud-based models. 
* **Phase 2 Solver:** GTO+ (via unofficial local socket interface using `customconnect.txt` on port 55143, potentially utilizing the `gto-` wrapper).

## System Architecture (Phase 1)
1.  **Game State Manager:** Tracks blinds, stacks, positions, hole cards, community cards, and betting history.
2.  **UI Layer:** Displays the table. Provides a text input box for the user's "Action" (e.g., Call, Raise $X, Fold) AND their "Reasoning".
3.  **Prompt Generator:** Takes the current Game State + User Action + User Reasoning and formats it into a structured prompt.
4.  **LLM Critic:** Receives the prompt, evaluates the play, and returns a critique to the UI layer.
